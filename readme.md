# Requisitos funcionais

[ ] Deve ser possível criar um usuário
[ ] Deve ser possível logar com o usuário
[ ] Deve ser possível registrar uma refeição, com nome, descrição, data, hora e se está dentro da dieta
[ ] Deve ser possível editar a refeição, alterando todos os dados
[ ] Deve ser possível apagar uma refeição
[ ] Deve ser possível listar todas as refeições de um usuário
[ ] Deve ser possível visualizar uma única refeição
[ ] Deve ser possível recuperar as métricas:
    [ ] Quantidade total de refeições registradas
    [ ] Quantidade total de refeições dentro da dieta
    [ ] Quantidade total de refeições fora da dieta
    [ ] Maior sequência de refeições dentro da dieta

# Regras de negócio

[ ] Para realizar as operações, o usuário deve estar logado
[ ] As operações devem ser vinculadas ao usuário, e não deve ser permitido interagir com operações de outro usuário
